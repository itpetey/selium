title: "Getting started"
description: "Run a local daemon, publish a contract, and launch a real Selium workload."
---

This guide gets you from zero to a running Selium workload on your machine. You will:

1. Build the current Selium runtime and CLI from this repo.
2. Create a local work directory and certificate bundle.
3. Start the runtime daemon.
4. Publish a contract and launch a real example module.

## Prerequisites

You will need stable Rust (see [rustup.rs](https://rustup.rs)), plus the WebAssembly target:

```bash
rustup target add wasm32-unknown-unknown
```

## 1. Build the local tools

From the repository root:

```bash
cargo build -p selium-runtime -p selium
```

That gives you two local binaries:

- `./target/debug/selium-runtime` for the daemon
- `cargo run -p selium -- ...` (or `./target/debug/selium`) for CLI workflows

## 2. Create a work directory and certificates

Selium keeps local certificates and guest modules under a single work directory:

```bash
mkdir -p .selium-local/modules .selium-local/certs
./target/debug/selium-runtime generate-certs --output-dir .selium-local/certs
```

The generated bundle gives the daemon a local server identity and gives the CLI a client certificate it can use for mTLS.

## 3. Start the daemon

In a dedicated terminal, start the current runtime daemon:


```bash
./target/debug/selium-runtime \
  --work-dir .selium-local \
  daemon \
  --listen 127.0.0.1:7100 \
  --cp-node-id local-node \
  --cp-public-addr 127.0.0.1:7100 \
  --cp-state-dir control-plane \
  --quic-ca certs/ca.crt \
  --quic-cert certs/server.crt \
  --quic-key certs/server.key \
  --quic-peer-cert certs/client.crt \
  --quic-peer-key certs/client.key
```

This starts the QUIC control endpoint, boots the embedded control-plane guest, and keeps a local node named `local-node` available for CLI operations.

In a second terminal, export the connection details once:

```bash
export SELIUM_DAEMON=127.0.0.1:7100
export SELIUM_NODE=local-node
export SELIUM_WORK_DIR=$PWD/.selium-local
export SELIUM_CERT_DIR=$SELIUM_WORK_DIR/certs
```

## 4. Run the `rpc-echo-service` example

This repository already includes current examples built around the present mental model. We'll use `examples/rpc-echo-service`, which exercises contract publication plus managed public event bindings.

Change into the example directory:

```bash
cd examples/rpc-echo-service
```

### Publish the contract package

The example checks in generated Rust bindings already, so the first required control-plane step is publishing the contract definition the runtime will use for its public endpoint names:

```bash
cargo run -p selium -- \
  --daemon-addr "$SELIUM_DAEMON" \
  --ca-cert "$SELIUM_CERT_DIR/ca.crt" \
  --client-cert "$SELIUM_CERT_DIR/client.crt" \
  --client-key "$SELIUM_CERT_DIR/client.key" \
  idl publish --input contracts/messaging.echo.v1.selium
```

### Build the guest module and copy it into the work dir

```bash
cargo build --manifest-path Cargo.toml --target wasm32-unknown-unknown
cp ../../target/wasm32-unknown-unknown/debug/rpc_echo_service.wasm \
  "$SELIUM_WORK_DIR/modules/"
```

### Launch the workload

This example uses direct operational launch: `selium start` plus managed endpoint bindings.

```bash
cargo run -p selium -- \
  --daemon-addr "$SELIUM_DAEMON" \
  --ca-cert "$SELIUM_CERT_DIR/ca.crt" \
  --client-cert "$SELIUM_CERT_DIR/client.crt" \
  --client-key "$SELIUM_CERT_DIR/client.key" \
  start \
  --node "$SELIUM_NODE" \
  --replica-key rpc-echo-demo \
  --event-reader echo.requested \
  --event-writer echo.requested \
  --event-reader echo.responded \
  --event-writer echo.responded \
  --module modules/rpc_echo_service.wasm
```

If the command succeeds, the daemon started the module, attached the contract-defined public endpoints, and the guest completed one typed round trip before idling.

## 5. Inspect and stop it

List the running replicas on the node:

```bash
cargo run -p selium -- \
  --daemon-addr "$SELIUM_DAEMON" \
  --ca-cert "$SELIUM_CERT_DIR/ca.crt" \
  --client-cert "$SELIUM_CERT_DIR/client.crt" \
  --client-key "$SELIUM_CERT_DIR/client.key" \
  list --node "$SELIUM_NODE"
```

Then stop the replica you just started:

```bash
cargo run -p selium -- \
  --daemon-addr "$SELIUM_DAEMON" \
  --ca-cert "$SELIUM_CERT_DIR/ca.crt" \
  --client-cert "$SELIUM_CERT_DIR/client.crt" \
  --client-key "$SELIUM_CERT_DIR/client.key" \
  stop --node "$SELIUM_NODE" --replica-key rpc-echo-demo
```

## What just happened

- `selium-runtime daemon` started the local runtime and control plane.
- `selium idl publish` registered the example's contract package.
- `selium start` launched one operational replica and injected the managed bindings for `echo.requested` and `echo.responded`.
- The guest used `selium_guest::io` against those bindings rather than manually building local queue topology.
