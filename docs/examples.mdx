Selium's examples are the fastest way to see the current platform in context. Most of them are guest-module examples built around `selium-guest`, checked-in `.selium` contracts, and generated Rust bindings. If you need host-side publish/subscribe or replay flows outside a guest module, start with [Rust SDK](./rust-sdk) instead.

The examples do not all use the same operational path: `control-plane-topology` is the full `idl publish` → `deploy` → `connect` → `agent --once` flow, most single-module examples run against a daemon with `selium start` / `selium stop`, and the network examples run directly in `selium-runtime` with shipped runtime configs and stop with `Ctrl-C`.

Start with the shared setup in [examples/README.md](https://github.com/itpetey/selium/tree/main/examples). If you only read one example first, read [control-plane-topology](https://github.com/itpetey/selium/tree/main/examples/control-plane-topology): it shows the current `idl publish` → `deploy` → `connect` → `agent --once` workflow end to end. Keep [Guest API](./applications) nearby for `selium-guest` reference details while you work through the guest examples.

## External-consumer follow-up

If you are using an example to seed a real topology and then hand Selium data to an external consumer:

- start with [control-plane-topology](https://github.com/itpetey/selium/tree/main/examples/control-plane-topology) for a concrete `idl publish` → `deploy` → `connect` → `agent --once` setup
- then read [CLI](./cli) for the external-consumer adoption cookbook covering inventory bootstrap, discovery, replay, and usage handoff
- use the CLI page's troubleshooting appendix when the bootstrap or resume cursor handoff is unclear
- keep [Rust SDK](./rust-sdk) nearby if the downstream consumer is a host-side Rust process rather than a guest module

## Choose by workflow

### Control-plane-managed workloads

- [control-plane-topology](https://github.com/itpetey/selium/tree/main/examples/control-plane-topology): publish `contracts/analytics.topology.v1.selium`, deploy three workloads, connect contract-defined endpoints, and reconcile them onto a node with `selium agent --once`. This is the best reference for the public `tenant/namespace/workload` and `tenant/namespace/workload#endpoint` model.

### Direct CLI launch against a running daemon

These examples use a running daemon plus `selium start` / `selium stop`, either with managed event bindings or with explicit `--module-spec` launch details.

#### Single-module managed-event flows

- [rpc-echo-service](https://github.com/itpetey/selium/tree/main/examples/rpc-echo-service): the smallest request/reply service built around contract-defined `echo.requested` and `echo.responded` endpoints.
- [event-broadcast](https://github.com/itpetey/selium/tree/main/examples/event-broadcast): one publisher, two independent readers, and a separate acknowledgement flow over managed event bindings.
- [pipeline-transform](https://github.com/itpetey/selium/tree/main/examples/pipeline-transform): a staged order-processing pipeline with ingress, reservation, and projection steps.
- [scatter-gather](https://github.com/itpetey/selium/tree/main/examples/scatter-gather): parallel request fan-out to multiple workers with result aggregation on a shared endpoint.

#### Custom launch and process lifecycle

- [typed-entrypoints](https://github.com/itpetey/selium/tree/main/examples/typed-entrypoints): custom entrypoints and typed launch arguments passed through `start --module-spec`, plus a managed `launch.recorded` binding.
- [process-supervisor](https://github.com/itpetey/selium/tree/main/examples/process-supervisor): parent/child orchestration with `selium_guest::process::ProcessBuilder` and contract-defined worker status events.
- [stateful-counter](https://github.com/itpetey/selium/tree/main/examples/stateful-counter): explicit checkpoint handoff plus a resume entrypoint for restartable guest logic.

### Direct runtime launch for guest networking

- [network-http-rpc](https://github.com/itpetey/selium/tree/main/examples/network-http-rpc): HTTPS request/response RPC over `selium_guest::network` with generated typed client/server helpers.
- [network-quic-stream](https://github.com/itpetey/selium/tree/main/examples/network-quic-stream): raw QUIC stream I/O over the guest network layer with generated typed stream helpers.

## Choose by guest API surface

- **Control-plane workflow**: [control-plane-topology](https://github.com/itpetey/selium/tree/main/examples/control-plane-topology)
- **Direct CLI launch with `start --module-spec`**: [typed-entrypoints](https://github.com/itpetey/selium/tree/main/examples/typed-entrypoints)
- **`selium_guest::process`**: [process-supervisor](https://github.com/itpetey/selium/tree/main/examples/process-supervisor), [typed-entrypoints](https://github.com/itpetey/selium/tree/main/examples/typed-entrypoints)
- **`selium_guest::network`**: [network-http-rpc](https://github.com/itpetey/selium/tree/main/examples/network-http-rpc), [network-quic-stream](https://github.com/itpetey/selium/tree/main/examples/network-quic-stream)
- **Managed event bindings and generated contract helpers**: [rpc-echo-service](https://github.com/itpetey/selium/tree/main/examples/rpc-echo-service), [event-broadcast](https://github.com/itpetey/selium/tree/main/examples/event-broadcast), [pipeline-transform](https://github.com/itpetey/selium/tree/main/examples/pipeline-transform), [scatter-gather](https://github.com/itpetey/selium/tree/main/examples/scatter-gather), [stateful-counter](https://github.com/itpetey/selium/tree/main/examples/stateful-counter), [control-plane-topology](https://github.com/itpetey/selium/tree/main/examples/control-plane-topology)
- **Restartable or stateful guest logic**: [stateful-counter](https://github.com/itpetey/selium/tree/main/examples/stateful-counter)

If you are looking for host-side Rust integrations instead of guest modules, use [Rust SDK](./rust-sdk) rather than the example index.
